Invoke the scan-first skill. Before building or spawning an agent on a coverage/effect/behaviour ticket, verify its claims — examples, counts, "this is still open", file/function locations — against live source: read the baseline metric, extract real volume and shape variants from the codebase, and trace one real case through the live pipeline to the line where it actually stops. Classifies each item OPEN / GHOST / PARTIAL so ghosts close by verification (no code) and agents spawn only on scan-verified-open, correctly-scoped lanes; stop a mis-scoped lane immediately. When orchestrating subagents, mark unverified claims as hypotheses and license the agent to reject the premise. Use before a coverage ticket, before spawning worktree agents on a wave, or when a backlog/audit table looks stale.
