Invoke the vibe-security skill. Audits the codebase for common security vulnerabilities introduced by AI coding assistants — exposed secrets, disabled database access control (Supabase RLS, Firebase rules), broken auth, missing rate limiting, insecure payment flows, mobile security gaps, and AI/LLM integration risks. Loads only the reference files relevant to the technologies in use. Produces severity-ranked findings (Critical → High → Medium → Low) with before/after code fixes. Use when asked to audit security, review for vulnerabilities, check if code is safe, or proactively when writing code that handles auth, payments, database access, API keys, or user data. Supports `--scope auth|payments|mobile|ai|data|db|deploy` to limit the audit to a specific area, and a path argument to target a specific directory. Codex trigger: `$vibe-security`.
