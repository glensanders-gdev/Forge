# Skill Health Report Format

Output inline and save to `~/.claude/knowledge/skill-health-report.md`:

```markdown
# Skill Health Report

**Generated:** YYYY-MM-DD
**Previous report:** YYYY-MM-DD (or "First check")
**Forge version:** [value from manifest.json forge_version]

---

## Health Scorecard

| Metric | Count | % |
|--------|-------|---|
| Total skills in manifest | N | — |
| Fully complete (all checks pass) | N | N% |
| Have SKILL.md | N | N% |
| Have command stub | N | N% |
| Have Failure Modes section | N | N% |
| Have Rules section | N | N% |

**Overall completeness:** N% (↑ improved / ↓ declined / — first check vs previous report)

---

## 🔴 Critical — Must Fix

### Manifest orphans (in manifest.json, no SKILL.md directory)
| Skill | Version | Action |
|-------|---------|--------|

### Missing required frontmatter
| Skill | Missing | Action |
|-------|---------|--------|

---

## ⚠️ Amber — Should Fix

### Missing Failure Modes section
| Skill | Action |
|-------|--------|

### Missing Rules section
| Skill | Action |
|-------|--------|

### Missing command stubs
| Skill | Action |
|-------|--------|

### CHANGELOG drift (version bumped, no CHANGELOG entry)
| Skill | Manifest version | Action |
|-------|-----------------|--------|

### Attribution gaps (origin: in frontmatter, no body credit)
| Skill | Origin | Action |
|-------|--------|--------|

---

## ℹ️ Informational

### Directory orphans (skills/ directory, not in manifest)
| Directory | Action |
|-----------|--------|

### Orphaned command stubs (commands/ file, not in manifest)
| Command | Action |
|---------|--------|

### Version mismatches (SKILL.md version != manifest version)
| Skill | SKILL.md | Manifest | Action |
|-------|----------|----------|--------|

### Forge installation
| Item | Value | Status |
|------|-------|--------|
| Version | v[N.N.N] | ✅ stamp present / ℹ️ stamp missing |
| Installed | YYYY-MM-DD | ✅ recent / ⚠️ [N] days ago — run /forge-update |
| Commit | [sha] | |

---

## ✅ Healthy

N skills pass all checks. (List omitted — view manifest.json for full inventory.)

---

## Recommended Actions

[Numbered, highest priority first. Each names the exact file or command to act on.]

---

*Generated by /skill-health — read-only audit, no files were modified.*
```

Omit sections with no findings — don't output empty tables.

If `--critical` flag was passed, show only 🔴 Critical and Recommended Actions.

If `--skill <name>` was passed, show the full breakdown for that skill only.
